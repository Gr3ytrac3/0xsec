I build and break systems to make them safer.

I’m an offensive security researcher and aspiring leader in kernel security — focused on kernel exploitation, vulnerability research, and red-team tradecraft. My path is self-directed and hands-on: I started in Computer Science & IT, spent years building full-stack software and honing leadership, product and go-to-market skills, then doubled down on the lowest layers of computing where real leverage lives.

Technically, I work with C/C++, x86_64/ARM64 assembly, Python and Bash. I debug with GDB/QEMU, run reproducible labs on QEMU/KVM, and use fuzzing and triage pipelines to hunt bugs. Practically, that means writing LKMs, prototyping PoCs, automating fuzzing/crash triage, reverse engineering, and producing reproducible writeups and tooling that others can run in safe lab environments.

I’ve shipped secure web apps, led team projects and hackathon builds, and I’m actively growing a portfolio of research: labs, open-source tools, and publishable PoCs. Current projects include a curated desktop hub for security tooling (The OffSec Desk). Kernel hardening experiments, and a repository of reproducible CVE demos.

My north star is clear: discover impactful flaws and weaknesses within the system, publish rigorous analysis, and build tools that shift how defenders and operators reason about system security. I'd love assist the next generation of researchers, contribute upstream fixes, and help teams harden critical infrastructure. In all, I keep on learning, adapting and upskilling to present the ideal profile of a true offensive security researcher.

If you’re working on kernel security, fuzzing, or need reproducible lab tooling — or if you want to collaborate on research or responsible disclosure — let’s connect.
